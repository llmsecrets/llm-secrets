# scrt4 v0.2 — architecture branch

> **You are looking at branch `architecture/v0.2.0`.** This is the long-lived branch where the v0.2 architecture is being built.
>
> **The v0.1.0 release on `main` is unaffected.** Anything in `daemon/bin/scrt4`, `Dockerfile.hardened`, `install/scrt4-docker.sh`, and the `docker-publish.yml` pipeline is untouched. Users running `curl -fsSL https://install.llmsecrets.com | sh` still get v0.1.0. This branch does not publish anything to Docker Hub or the install URL.

## What's in this branch

A parallel implementation of the v0.2 architecture defined in [`docs/ARCHITECTURE-V0.2.md`](ARCHITECTURE-V0.2.md):

| File | Purpose |
|---|---|
| `daemon/bin/scrt4-core` | Trusted-computing-base CLI (auth gates, vault protocol handoff, dispatcher, core commands) with `# TCB:` annotations |
| `daemon/bin/scrt4-modules/*.sh` | Opt-in modules: `share` is a full reference impl; the rest are stubs that register their commands and direct users to the v0.1.0 monolith |
| `modules.manifest` | Declares which modules each distribution (hardened/core-only) includes |
| `scripts/build-scrt4.sh` | Reads the manifest, validates module headers, concatenates core + selected modules into a single bash binary |
| `docs/ARCHITECTURE-V0.2.md` | Full architecture explanation |
| `docs/TCB.md` | Trusted computing base inventory + invariants |
| `docs/FORMAL-VERIFICATION-INPUT.md` | Handoff doc for the verification team |

## What it does NOT do

- **Does not modify `daemon/bin/scrt4`.** The v0.1.0 monolith is preserved verbatim.
- **Does not modify `Dockerfile.hardened`.** Hardened release pipeline is untouched.
- **Does not modify `install/scrt4-docker.sh`.** Live install URL unchanged.
- **Does not push images to Docker Hub.** No GitHub Actions wired into this branch.

## Try it locally

```sh
# Make sure you're on the architecture branch
git checkout architecture/v0.2.0

# Build both v0.2 distributions (no daemon needed for this)
scripts/build-scrt4.sh hardened   /tmp/scrt4-hardened-v2
scripts/build-scrt4.sh core-only  /tmp/scrt4-core-only

# Each output is a self-contained bash script
wc -l /tmp/scrt4-hardened-v2 /tmp/scrt4-core-only

# Confirm core-only really has no modules (share returns "Unknown command")
/tmp/scrt4-core-only share
```

## TCB inventory at a glance

```sh
# Every TCB-marked function in the v0.2 core
grep -n '^# TCB:' daemon/bin/scrt4-core
```

Should produce ~3-4 entries (auth gates and the secret-injection helper). The full inventory across the Rust daemon and CLI is in [`docs/TCB.md`](TCB.md). The `// TCB:` annotations in the Rust daemon are not yet added — that's part of issue #60.

## Why a long-lived branch instead of merging incrementally to main

1. **Stability of v0.1.0.** Every step of the v0.2 work could destabilize the monolith if it landed on main piecemeal. Keeping it parallel lets users continue to run v0.1.0 without surprises.
2. **Reviewability.** The v0.2 architecture is a single coherent change. Reviewing it as one branch — read the docs, then the core, then the build script, then the modules — is much easier than reviewing 15 small PRs that each only make sense in the context of the others.
3. **Reversibility.** If we decide we don't like the module system, abandoning the branch costs nothing. Nothing on main has to be unwound.
4. **Formal verification cadence.** The v2 hardened verification effort wants a stable target. A long-lived branch lets the verification team fork it cleanly when they're ready.

## When this branch becomes main

The plan is **not** to merge this branch as-is. The expected sequence:

1. Iterate on this branch until the architecture, build script, manifest, and core API are settled.
2. Port the remaining modules (currently stubs) one by one, on PRs to this branch.
3. Add a CI matrix to this branch that builds + smoke-tests every distribution.
4. Once the v0.2 hardened build is byte-equivalent (or behavior-equivalent) to the v0.1.0 monolith, open a single squash-merge PR to main and tag v0.2.0.
5. After the squash, delete this branch.

Until step 4, **only this branch changes**. Main and the published artifacts stay frozen at v0.1.0.
